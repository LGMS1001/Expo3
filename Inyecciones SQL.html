SQL Injections

SQL injection is a method of infiltration of intrusive code that uses a computer vulnerability present in an application at the input validation level to perform operations on a database.

The origin of the vulnerability lies in the incorrect verification or filtering of the variables used in a program that contains or generates SQL code. It is, in fact, a bug from a more general class of vulnerabilities that can occur in any programming language or script that is embedded in another.

It is known as SQL Injection, indistinctly, to the type of vulnerability, the method of infiltration, the fact of embedding intrusive SQL code and the portion of the embedded code.

A successful injection attack could spoof identities, create new identities with admin rights, access all data on the server, or destroy/modify data to make it unusable.

Blind SQL injection

Blind SQL injection is a function and attack technique that uses SQL injection. It is evidenced when a web page, due to a security flaw, does not display error messages as there are no correct results when querying the database, always displaying the same content (that is, there is only a response if the result is correct).
