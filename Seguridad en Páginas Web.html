Web Page Security

The purpose of web security is to prevent attacks of any kind. More formally, security is the action/practice of protecting websites from unauthorized access, use, modification, destruction or interruption.

Effective website security requires design efforts throughout the entire website: in your web application, in your web server configuration, in your policies for creating and renewing passwords, and in your client-side code.

Website security threats


Cross Site Scripting (XSS)

XSS is a term used to describe a class of attacks that allow the attacker to inject client-side scripts, through the website, into other users' browsers. Since the injected code goes from the site's server to the browser, it is assumed to be trusted, and hence can do things like send the attacker the authorization cookie to the user's site. Once the attacker has the cookie they can log into the site as if they were the real user and do anything the real user can do. Depending on which site it is, this could include accessing your credit card details, viewing contact details or changing passwords, etc.

The best defense against XSS vulnerabilities is to remove or disable any tags that might contain instructions to execute code. In the case of HTML this includes tags like "script", "object", "embed", y "link".

The process of modifying user data in such a way that it cannot be used to run scripts or otherwise affects the execution of server code is known as "input sanitization."
